# 数据供给与分级策略（v1）

## 1. 原则
- **可控优先**：按配额供给，不做无限量数据开放。
- **脱敏优先**：优先提供聚合特征和标签，减少原始文本暴露。
- **覆盖优先**：采样追求“场景覆盖”，不是“样本越多越好”。
- **可回归优先**：每轮迭代保留固定评测集与回放集，防遗忘。

## 2. 数据分级（Data Contract）

### P0（可下发训练包）
适合学生项目直接使用：
- 商品基础信息（按采样）
- 成分知识（结构化）
- 合规规则（公开）
- 评论标签/摘要（受控）

### P1（受限访问）
仅提供特征或 API 访问，不直接导出原始全量：
- 热点趋势原始流
- 细粒度行为特征
- 高敏感商业字段

### P2（盲测保留包）
仅用于最终验收，不进入训练：
- 保密评测问题集
- 新近业务样本

## 3. 每轮配额建议（2周）
- SKU：800–1500（推荐 1000）
- 评论：10000–30000（推荐 20000，优先近90天）
- 趋势词：200–500（推荐 300）
- 成分条目：300–500
- 合规规则：全量同步

## 4. 商品采样规则（分层抽样）
采样维度：
- 品类：护肤/彩妆/清洁/防晒
- 价格带：低/中/高
- 品牌层级：头部/腰部/长尾
- 生命周期：新品/稳定/衰退
- 功效：保湿/修护/控油/抗老/美白等

推荐配比：
- 头部稳定款：40%
- 腰部主销款：35%
- 长尾探索款：15%
- 新品趋势款：10%（或单独按趋势补样）

## 5. 趋势补样规则
在分层抽样基础上追加趋势补样：
- 触发条件：7日增长率 > 30% 或进入周榜 Top N。
- 补样策略：本轮样本中 20% 来自“高增长趋势词关联商品”。

## 6. 用户画像替代策略（无全量用户画像时）
可使用“弱画像”替代：
- 来源：站内聚合行为 + 小红书趋势信号 + 公开内容标签。
- 画像字段：肤质桶、预算桶、功效偏好桶、风险敏感桶。
- 禁止直接输出可识别个人身份字段。

## 7. 数据开放边界
- 禁止输出：手机号、账号ID映射、可逆身份信息。
- 控制输出：评论原文只给抽样，优先提供摘要/关键词/风险标签。
- 使用约束：仅限课程研究，不得二次分发或商用。

## 8. 交付物
每轮需交付：
- `sampling_report`：采样覆盖率、配比、时间窗口。
- `data_quality_report`：完整率、重复率、异常率。
- `release_manifest`：本轮可用数据包清单与版本号。
