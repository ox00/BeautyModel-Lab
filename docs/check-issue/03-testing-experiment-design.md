# Task Package - Testing And Experiment Design

## Goal
- Build the shared validation baseline for both technical quality and business usefulness.
- Freeze the first MVP benchmark set and test method.

## Scope
- technical test cases
- business test cases
- benchmark question set
- holdout design
- experiment comparison table

## Deliverables
- `MVP` question set
- technical test checklist
- business test checklist
- evaluation template
- experiment log format

## Technical Test Focus
- loader correctness
- retrieval correctness
- evidence grounding quality
- compliance hit or miss behavior
- trend freshness behavior

## Business Test Focus
- usefulness of skincare explanation
- usefulness of recommendation logic
- ability to balance science and trend
- clarity of risk reminders
- readability and trustworthiness

## Acceptance Criteria
- The benchmark set covers all four MVP question types.
- Technical and business tests use the same frozen answer format.
- The experiment sheet can compare at least two versions of the system.

## Dependencies
- Depends on Engineering A and B producing a runnable pipeline.
- Should coordinate closely with architecture and domain owner on test case wording.

## Out Of Scope
- long-form final report writing
- external user study at this stage
