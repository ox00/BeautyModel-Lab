# 实验设计与里程碑（v1）

## 1. 实验目标
验证两类关键能力：
- 数据维度与采样策略是否提升推荐稳定性与覆盖。
- 蒸馏飞轮是否在有限数据下实现持续改进。

## 2. 基线对比设计
- Baseline-A：规则/检索基础版（无蒸馏）
- Baseline-B：一次性蒸馏版（无多轮回流）
- Method-C：飞轮版（四步飞轮 + 回放机制）

## 3. 指标体系
- Correctness
- Completeness
- Safety
- Timeliness
- P95 Latency
- 复发率（已修复问题重复出现比例）

## 4. 消融实验建议
- Ablation-1：去掉趋势补样
- Ablation-2：去掉规则校验
- Ablation-3：去掉回放机制
- Ablation-4：只做单轮蒸馏

## 5. 里程碑（建议）
- M1（第1-2周）：
  - 完成数据字典落地、首轮采样、Baseline-A/B
- M2（第3-4周）：
  - 跑通 Method-C 第一轮，提交 eval/risk 报告
- M3（第5-6周）：
  - 完成至少一次误差回流与二轮迭代
- M4（第7-8周）：
  - 完成消融实验与结题汇报材料

## 6. 每周最小交付
- 周报：数据供给变化 + 样本规模 + 风险事件
- 产物：采样报告、质量报告、评测报告、复盘报告
- 决策：是否满足放行门槛、是否进入下一轮

## 7. 汇报建议（面向导师/团队）
- 一页图说明“数据维度 + 飞轮机制”
- 一页表展示“基线 vs 飞轮”关键指标变化
- 一页风险复盘（问题->归因->修复->结果）
