# 技术方案：数据维度与采样设计（v1）

## 1. 背景与目标
在企业数据不能全量开放的前提下，本方案目标是：
- 用有限且合规的数据，支撑美业推荐场景的 MVP 开发与评估。
- 保证数据对模型训练“够用且可解释”，并可持续迭代。
- 为团队和导师提供可审计的采样逻辑，而非临时拼接样本。

## 2. MVP 必备数据维度

### 2.1 用户弱画像维度（不依赖全量个人画像）
- 核心字段：肤质桶、预算桶、诉求桶（修护/控油/抗老等）、风险敏感桶。
- 数据来源：站内聚合行为、趋势标签映射、任务问卷（匿名化）。
- 作用：作为推荐条件，不存储可识别个人身份信息。

### 2.2 商品维度
- 核心字段：SKU、品牌、品类、价格带、成分、功效标签、生命周期（新品/稳定/衰退）。
- 数据来源：企业商品库 + 合规渠道公开信息。
- 作用：构成候选集与排序特征基础。

### 2.3 成分知识维度
- 核心字段：成分功效、适配肤质、风险级别、搭配禁忌、证据等级。
- 数据来源：法规、词典、期刊、专家知识。
- 作用：为“科学护肤解释”提供可追溯依据。

### 2.4 趋势热点维度
- 核心字段：关键词热度、7日增速、平台来源、关联成分/类目。
- 数据来源：小红书/电商等趋势信号（合规接入）。
- 作用：支撑动态更新与趋势推荐能力。

### 2.5 口碑反馈维度
- 核心字段：情感极性、风险词、功效提及、时间分布。
- 数据来源：评论数据（可摘要化/标签化输出）。
- 作用：用于效果验证、风险识别、候选重排序。

### 2.6 合规规则维度
- 核心字段：禁用词、敏感描述、处理动作（block/warn/replace）。
- 数据来源：公开规范与公司规则。
- 作用：训练与推理阶段双重守门。

## 3. 采样策略（分层抽样 + 趋势补样）

### 3.1 分层抽样
按以下维度同时分层，避免样本单一：
- 品类：护肤/彩妆/清洁/防晒
- 价格带：低/中/高
- 品牌层级：头部/腰部/长尾
- 生命周期：新品/稳定/衰退
- 功效标签：保湿/修护/控油/抗老/美白等

建议配比（可按迭代微调）：
- 头部稳定款 40%
- 腰部主销款 35%
- 长尾探索款 15%
- 新品趋势款 10%

### 3.2 趋势补样
在分层样本外增加趋势样本：
- 触发条件：7日增速 > 30% 或进入周榜 Top N。
- 补样比例：最终样本中约 20% 来自趋势关联商品。

## 4. 数据预算与供给边界
- 每两周预算建议：SKU 800–1500、评论 1万–3万、趋势词 200–500、成分 300–500。
- 数据分级：
  - P0：可下发训练包
  - P1：受限访问（特征/API）
  - P2：盲测保留包
- 边界要求：禁止个人可识别信息，评论优先摘要/标签化。

## 5. 质量控制与偏差治理
- 质量门槛：完整率 >=95%，重复率 <=2%，趋势新鲜度 <=7天。
- 偏差检查：
  - 品类、价格带、品牌层级分布偏差
  - 新品过采样导致的稳定性偏差
  - 平台来源偏差
- 治理措施：配额回调、分层重采样、渠道平衡。
