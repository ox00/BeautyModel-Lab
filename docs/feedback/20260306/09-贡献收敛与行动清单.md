## 2. 收敛理由

### 2.1 为什么砍掉"蒸馏飞轮"作为核心贡献？

| 因素 | 说明 |
|------|------|
| **时间约束** | 8周完成多轮迭代不现实，最多跑1-2轮 |
| **验证难度** | "飞轮"效果需要长期数据，课程项目无法证明 |
| **贡献归属** | 蒸馏本身不是新方法，只是应用 |
| **风险** | 如果蒸馏效果不明显，整个贡献失效 |

**解决方案**: 保留"单轮蒸馏验证"作为工程实现，不作为研究贡献。

### 2.2 为什么保留"TrendScore"？

| 优势 | 说明 |
|------|------|
| **可验证** | 与人工标注对比，有量化指标 |
| **可复现** | 公式明确，他人可以实现 |
| **有创新** | 将模糊的"趋势"概念可操作化 |
| **8周可完成** | Feasibility + 正式实验足够 |

### 2.3 为什么保留"冲突解决策略"？

| 优势 | 说明 |
|------|------|
| **问题明确** | Feasibility证明冲突真实存在 |
| **可消融** | 去掉该模块性能下降 = 贡献成立 |
| **实用价值** | 导师看重业务落地 |
