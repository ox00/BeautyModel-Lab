# BeautyModel 数据支持包（v1）

## 目的
本目录用于支持“美业消费者推荐（科学护肤 + 流行趋势）”方向的课程项目落地，重点解决以下问题：
- 在企业数据不能全量开放的前提下，保证项目可做、可评估、可复现。
- 统一团队对数据输入、更新节奏、评测门槛的理解。
- 形成“有限数据供给 + 增量蒸馏 + 回归评测”的工程闭环。

## 文档清单
- `01-数据供给与分级策略.md`：数据供给边界、分级、配额与采样规则。
- `02-数据字典模板.md`：核心表结构与字段标准。
- `03-两周迭代SOP.md`：两周一个版本的标准执行流程。
- `04-评测与发布门槛.md`：数据质量门槛、模型门槛、灰度门槛。
- `05-data_budget.yaml.example`：每两周数据供给预算示例（可直接复制改名使用）。
- `06-sampling_report-template.md`：采样覆盖与偏差说明模板。
- `07-data_quality_report-template.md`：数据质量检查与放行模板。

## 使用方式（建议）
1. 每轮迭代先锁定 `01` 中的配额和采样范围。
2. 数据接入按 `02` 字段标准落表。
3. 开发与训练按 `03` 时间节奏推进。
4. 发布前必须满足 `04` 的全部硬门槛。

## 适用范围
- 场景A：肤质/诉求到成分与产品推荐。
- 场景B：趋势词到同款/替代方案推荐（可作为 phase2 扩展）。
