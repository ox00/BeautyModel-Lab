一、📄 需求报告（Software Requirements Specification）
1. 项目概述
1.1 项目名称

Multi-Agent Trend Crawling System (基于 MediaCrawler)

1.2 项目背景

随着社交媒体平台（如小红书、抖音、B站等）的内容爆炸式增长，趋势信息（trend signals）对市场分析、内容创作、商业决策具有重要价值。

现有开源项目 MediaCrawler 已实现多平台内容爬取，但缺乏：

自动化趋势驱动
智能数据处理
多Agent协作机制
用户友好界面
1.3 项目目标

在 MediaCrawler 基础上，开发一个：

基于 Multi-Agent 的智能趋势内容爬取与分析系统

实现：

自动关键词驱动爬取
多平台统一管理
AI数据清洗
JSON结构化输出
2. 系统总体描述
2.1 系统架构

系统采用分层架构：

Frontend（后续开发）
        ↓
Backend API（FastAPI）
        ↓
Multi-Agent Layer
        ↓
MediaCrawler（核心爬虫引擎）
        ↓
Database / Storage
2.2 关键设计原则
❗ 不重写爬虫 → 复用 MediaCrawler
模块解耦（Agent化）
可扩展（支持更多平台/AI能力）
可自动化运行
3. 功能需求（Functional Requirements）
3.1 用户账号管理
功能描述

系统应支持多个平台登录信息的管理。

具体要求
存储平台账号信息：
platform（平台）
cookies / token
状态（active / expired）
支持：
添加账号
更新账号
删除账号
多账号轮换（防封）
3.2 Trend Signal（关键词库管理）
功能描述

用于驱动爬虫任务的核心数据源。

功能要求
添加关键词
设置：
平台
优先级
抓取频率

示例：

{
  "keyword": "露营装备",
  "platform": "xiaohongshu",
  "frequency": "daily",
  "priority": 0.9
}
3.3 自动爬虫调度
功能描述

系统自动定期执行爬虫任务。

功能要求
定时执行（cron）
按关键词触发
按平台分发任务
3.4 多平台内容爬取（复用 MediaCrawler）
功能描述

通过调用 MediaCrawler 实现数据抓取。

功能要求
支持：
小红书
抖音
B站（取决于 MediaCrawler支持）
支持：
关键词搜索
返回原始数据
3.5 数据清洗（AI处理）
功能描述

对爬取数据进行结构化处理。

功能要求
去噪（广告/无关内容）
提取摘要
主题分类
情感分析
输出格式
{
  "summary": "...",
  "topics": ["露营", "户外"],
  "sentiment": "positive"
}
3.6 数据存储
功能描述

存储原始与处理后数据。

要求
原始数据（raw_data）
清洗数据（cleaned_data）
时间戳
来源平台
3.7 JSON输出接口
功能描述

提供统一数据输出接口。

示例：
{
  "keyword": "露营",
  "platform": "douyin",
  "results": [...]
}
4. 非功能需求（Non-Functional Requirements）
4.1 性能
支持并发爬取
支持任务队列
4.2 可扩展性
可增加新平台
可增加新Agent
4.3 安全性
账号信息加密存储
API鉴权（可选）
4.4 可维护性
模块化设计
清晰日志系统
5. Multi-Agent设计
Agent划分
Agent	功能
Trend Agent	管理关键词
Scheduler Agent	调度任务
Crawler Agent	调用 MediaCrawler
Cleaning Agent	AI数据清洗
Insight Agent（可选）	趋势分析
Agent通信方式
Redis Queue / Celery
或简单任务调度器
