# 导师沟通提问清单（Proposal 评审会用）

## 0. 会议目标
在本次沟通中，尽量完成“可执行拍板”：
- 研究问题范围冻结
- Baseline 与指标冻结
- 第一阶段里程碑与交付物冻结

---

## 1. 研究问题与范围（必须拍板）
- [ ] 我们是否以“科学护肤正确性 + 趋势响应速度”作为双目标主问题？
- [ ] 第一阶段是否只做单场景 MVP（科学护肤问答为主，趋势推荐为辅）？
- [ ] 是否同意将“多轮对撞飞轮”作为核心方法贡献之一？
- [ ] Proposal 中是否需要保留“用户价值线 + 技术方案线”双线论证？

**希望导师明确：**哪些内容属于“必须做”，哪些是“可选增强”。

---

## 2. Baseline 与评估（必须拍板）
- [ ] Baseline-A：仅科学知识问答（无趋势机制）是否可作为主对照？
- [ ] Baseline-B：仅趋势推荐（无科学约束）是否可作为主对照？
- [ ] Baseline-C：通用大模型问答（无专门管线）是否保留？

- [ ] 指标集合是否认可：Correctness / Completeness / Safety / Trend Freshness / Latency / Explainability？
- [ ] 是否要求做消融实验（去 Trend、去合规、去回放）？
- [ ] 课程答辩最低实验强度：需要几轮迭代、几组对照？

**希望导师明确：**最低可通过标准（例如至少 2 个 baseline + 1 个消融）。

---

## 3. 数据与合规边界（必须拍板）
- [ ] 是否认可 P0/P1/P2 分级供给机制（有限预算、不全量开放）？
- [ ] 是否认可“弱画像”替代真实用户画像（匿名聚合）？
- [ ] 是否认可评论以摘要/标签优先，不开放大规模原文？
- [ ] 合规边界是否需要单独章节（伦理与安全声明）？

**希望导师明确：**课程项目对企业数据合规披露的最低要求。

---

## 4. 方法设计关键点（建议拍板）
- [ ] “Trend 定义函数”是否需要形式化写入 Proposal（热度、增速、持续时间、跨平台一致性）？
- [ ] 趋势与科学冲突时，是否采用“科学安全优先”的决策规则？
- [ ] multi-agent 数据采集是否作为工程实现，不作为理论主创新？
- [ ] 我们的创新主轴是否应聚焦在：
  - 双目标协同框架
  - 趋势可操作化定义与更新
  - 蒸馏飞轮与回流机制

---

## 5. 里程碑与交付物（必须拍板）
- [ ] M1-M4 的阶段划分是否合理（8周）？
- [ ] 每阶段最低交付是否认可：
  - 文档：数据方案、方法设计、评估协议
  - 系统：可演示原型
  - 实验：对照与消融结果
- [ ] 最终答辩最看重的是“方法创新”还是“工程完整度”？

**希望导师明确：**评分权重（方法/实验/工程/展示）。

---

## 6. 风险与预案（建议确认）
- [ ] 若趋势数据质量不稳定，是否允许降级为“周更新”而非“准实时”？
- [ ] 若多模型成本超预算，是否允许收敛到单 provider + 小模型辅助？
- [ ] 若时间不足，优先保留哪些模块（建议：评估闭环 > 多模态扩展）？

---

## 7. 会后输出（我们承诺）
会后 24 小时内输出：
1. 冻结版 Proposal 大纲（含导师反馈修改）
2. 冻结版 baseline 与指标清单
3. 第一阶段任务分解与负责人列表
4. 下次汇报时间点与检查项
