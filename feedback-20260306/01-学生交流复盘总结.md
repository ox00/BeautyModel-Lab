# 学生交流复盘总结

## 1. 本次讨论后我们对课题的统一定义
我们聚焦的课题是：

> 基于科学理性护肤 + Trending 审美的 QA 对话顾问。

它不是单一问答系统，而是“专家顾问 + 情报采集解读”的复合体系。

- 科学理性护肤：保障建议的正确性、可解释性与安全性。
- Trending 审美：保障建议对流行变化的及时响应。

## 2. 我们对导师关注重点的理解
我们理解导师当前重点，不是“做一个功能”，而是验证这是一个可研究、可评估、可改进的技术问题。核心关注点包括：

1) 问题是否成立：现有方案为何难以同时兼顾“专业正确性”和“趋势响应速度”。
2) 改进是否清晰：我们相对于 baseline 的具体改进点是什么。
3) 方法是否可落地：在课程周期内，数据、流程、模型、更新机制是否可完成。
4) 评估是否严谨：是否有对照组、指标和消融实验。
5) 价值是否闭环：技术价值与用户价值是否都能证明。

## 3. 我们当前形成的核心研究问题
### RQ1：如何在多数据源条件下实现准实时更新，同时保证数据质量与一致性？
- 涉及维度：IP 热度、新品、平台趋势、口碑反馈等。
- 方向：基于 multi-agent 的数据采集与质量守门。

### RQ2：如何定义 Trend 并将其转化为可执行推荐能力？
- 关键问题：什么算 Trend、如何动态更新、如何反向驱动数据源更新。
- 目标：将 Trend 信号与科学护肤知识融合，而非相互冲突。

## 4. 我们拟采用的方法主线
### 方法主线A：数据维度与采样机制
- 建立 MVP 必备维度：商品、成分、趋势、口碑、合规、弱画像。
- 采用分层抽样 + 趋势补样，确保“覆盖性 + 时效性”。

### 方法主线B：蒸馏与多轮对撞飞轮
- 四步飞轮：Teacher 生成 -> 规则/知识校验 -> Student 蒸馏 -> 误差回流。
- 配套机制：回放集与回归门槛，控制遗忘。

## 5. “参考分析 + 价值提炼”如何落地
我们将从两条线完成导师要求的对比分析：

1) 场景方案线：
- 对比现有第三方方案在“专业性/趋势性/更新能力”上的不足。
- 提炼我们的 improve（双目标协同而非单点优化）。

2) 用户价值线：
- 对比消费者在“跟流行节奏 + 科学护肤”上的实际痛点。
- 提炼我们在解释性、安全性、时效性上的产品创新。

## 6. 当前阶段性结论
- 课题方向明确且具业务适配性。
- 技术重点已聚焦到“数据更新机制 + Trend 机制 + 飞轮迭代”。
- 下一步应进入正式 Proposal 文稿与评估设计确认阶段。
