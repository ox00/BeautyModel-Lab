# Proposal 中文母稿大纲（AIBA 毕业实践项目）

## 0. 标题（建议）
**Beauty QA Advisor: 面向科学护肤与趋势审美协同优化的多智能体对话顾问系统**

## 1. 研究背景与问题定义
### 1.1 背景
- 美业消费者决策同时受“科学护肤知识”和“社媒趋势审美”影响。
- 现有系统通常偏向其中一端：
  - 科学导向系统：专业但更新慢。
  - 趋势导向系统：更新快但专业性和安全性不足。

### 1.2 问题定义
我们要解决的问题是：

> 如何构建一个既具科学可靠性、又具趋势响应能力的 QA 对话顾问系统？

### 1.3 研究目标
- 在准确性、安全性、解释性前提下提升趋势响应能力。
- 在有限可用企业数据下建立可持续迭代机制。

## 2. 相关工作与问题缺口（Related Work & Gap）
### 2.1 场景方案线（技术对比）
- 现有方案类型、优势与短板。
- 关键缺口：双目标冲突（专业正确性 vs 趋势时效性）。

### 2.2 用户价值线（产品对比）
- 现有消费者工具在“跟趋势 + 科学护肤”方面的体验缺口。
- 关键缺口：建议缺乏可解释性、安全边界不清晰。

### 2.3 我们的改进定位
- 双通道协同：Science Channel + Trend Channel。
- 合规与风险作为系统内生约束而非事后补丁。

## 3. 方法与系统设计（Method）
### 3.1 输入输出定义
- Input：用户问题、肤质/诉求、预算、偏好、可选趋势关键词。
- Output：推荐建议、科学依据、替代方案、风险提示、免责声明。

### 3.2 数据流定义
- 多源采集 -> 标准化 -> 趋势识别 -> 检索/推理 -> 合规过滤 -> 输出。

### 3.3 核心组件（Components）
- Data Ingestion Agents（多源采集）
- Trend Engine（趋势识别与更新）
- Science Knowledge Layer（成分与法规知识）
- Dialogue Orchestrator（会话编排）
- Safety/Compliance Gate（安全合规审查）
- Distillation Flywheel（持续学习）

## 4. 核心难点与技术路线
### 4.1 难点A：多数据源准实时更新
- 研究问题：如何保证 freshness 同时维持质量一致性？
- 路线：multi-agent 采集 + 数据质量门槛 + source fallback。

### 4.2 难点B：Trend 发现与动态更新
- 研究问题：如何定义 Trend 并转化为推荐能力？
- 路线：热度增速、跨平台一致性、持续时间、新品关联度等综合判定。

### 4.3 难点C：趋势与科学知识冲突处理
- 研究问题：当 Trend 与科学约束冲突时如何决策？
- 路线：优先级规则 + 风险打分 + 解释性输出。

## 5. 创新点（Contributions）
1) 双目标协同的 QA 顾问框架（专业性 + 趋势性）。
2) 面向美业的 Trend 可操作化定义与触发更新机制。
3) 生成-校验-蒸馏-回流的多轮对撞飞轮。
4) 面向课程实践的可复现实验协议（baseline + 消融 + 回归）。

## 6. 实验设计与评估方案
### 6.1 Baselines
- Baseline-A：仅科学知识问答（无趋势机制）
- Baseline-B：仅趋势推荐（无科学约束）
- Baseline-C：通用大模型问答（无专门管线）

### 6.2 评价指标
- Correctness（专业正确性）
- Completeness（回答完整性）
- Safety（合规/安全）
- Trend Freshness（趋势时效）
- Latency（响应效率）
- Explainability（可解释性）

### 6.3 消融实验
- 去掉 Trend Engine
- 去掉 Compliance Gate
- 去掉回放机制
- 去掉多轮回流

## 7. 数据方案与合规边界
### 7.1 数据维度（MVP 必备）
- 商品、成分、趋势、口碑、合规规则、弱画像。

### 7.2 数据供给策略
- P0/P1/P2 分级供给。
- 分层抽样 + 趋势补样。
- 有限预算可复用迭代。

### 7.3 合规边界
- 去标识化、最小必要原则、用途限制。

## 8. 项目计划与里程碑
- M1（1-2周）：问题定义冻结 + baseline 搭建
- M2（3-4周）：多源采集 + Trend 机制 + 首轮实验
- M3（5-6周）：飞轮二轮迭代 + 消融实验
- M4（7-8周）：结果汇总 + 正式答辩材料

## 9. 风险与应对
- 数据噪声高 -> 质量门槛 + 抽检
- 生成幻觉 -> 规则校验 + 知识一致性检查
- 灾难性遗忘 -> 回放集 + 回归门槛
- 范围过大 -> MVP 场景冻结

- 方法总结（可复现技术文档）
