# Proposal 补强清单（对齐 SES 样本标准）

## A. 必补内容（提交前）
- [ ] 相关工作对比表（至少 4–6 个方案，含 strengths/limitations/our improve）
- [ ] 明确 RQ 与 Hypotheses（RQ1/2/3 + H1/2/3）
- [ ] TrendScore 指标定义与阈值触发规则
- [ ] Feasibility 小实验（30–50 条）
- [ ] 实验样本规模、标注流程、统计方法
- [ ] 参考文献补齐（格式统一，删除空项）

## B. 建议图表（导师阅读友好）
- [ ] Figure 1: 系统总架构（组件与数据流）
- [ ] Figure 2: 趋势更新与飞轮闭环图
- [ ] Table 1: Related work vs Our improve
- [ ] Table 2: Baselines / Metrics / Ablation matrix
- [ ] Table 3: Milestone & deliverables

## C. 评估设计最低标准
- [ ] 3 个 baseline
- [ ] 3 个 ablation
- [ ] 固定回归集 + 动态趋势集
- [ ] 至少 2 轮迭代结果

## D. 文稿质量控制
- [ ] 全文术语一致（Trend、Safety、Explainability 等）
- [ ] 每节均有“可验证结论”而非概念描述
- [ ] 避免口号式表述（如“创新”必须落在可测指标）
- [ ] 图表编号、引用、参考文献统一

## E. 组内分工建议（可直接开会用）
- [ ] 文献与对比：1人
- [ ] 方法与架构图：1人
- [ ] 数据与合规：1人
- [ ] 实验与指标：1人
- [ ] 文稿整合与排版：1人

## F. 与导师确认的拍板项
- [ ] MVP 范围（单场景 or 双场景）
- [ ] 评分重点（方法创新 vs 工程完整度）
- [ ] 最低可通过实验强度
- [ ] 是否要求显著性检验
